# ID: P0-03
## Title
Add toolbox route group + tool registry scaffold in `src/`

## Context
- Current app under `src/app/*` is a portfolio (home sections, `/services`, `/case-studies`, `/qualification`).
- A toolbox platform exists only as a reference in `_ops/external/asdev-persiantoolbox/app/(tools)/*` plus registry/SEO helpers in `_ops/external/asdev-persiantoolbox/lib/tools-registry.ts`.
- Tool SEO and sitemap patterns exist in `_ops/external/asdev-persiantoolbox/app/sitemap.ts` and `_ops/external/asdev-persiantoolbox/components/seo/ToolSeoContent.tsx`.

## Steps
1. Create a new route group in `src/app/(tools)/...` (or equivalent) without deleting/renaming existing portfolio routes.
2. Add a `src/lib/tools-registry.ts` that defines:
   - categories (PDF/Image/Text/Validation/Date/Finance)
   - a normalized tool list (id/path/title/description/keywords/indexable/lastModified/content)
3. Add a tool hub page (e.g., `/tools`) and category pages (e.g., `/pdf-tools`, `/image-tools`) with Persian-first copy.
4. Add a consistent “Tool layout” wrapper for RTL, breadcrumbs, and inline help blocks.
5. Add placeholder tool pages for the initial MVP list (even if the tool implementations land in later tasks).

## Acceptance criteria
- `/tools` renders a list of categories and “popular tools” with Persian titles/descriptions.
- `/pdf-tools` and `/image-tools` pages render with consistent layout and internal links.
- Each tool path in registry has a corresponding route (at least stub pages).
